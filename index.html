Use Single-Page Application (SPA)

SPA = web application that lives within a single HTML page Application views are injected into and removed from DOM as needed Advantages are: 1. Absence of page refreshes => more fluid and closer to native experience 2. UI is entirely created at client-side: ideal architecture for applications that work offline.

Let's Code! Use SPA

zie ook op: http://bit.ly/1d3VrYr 1. continu to use Employees Database Directory example 2. in index.html: remove the HTML markup inside the body tag (with the exception of the script tags). 3. inside the immediate function in app.js, define a function named renderHomeView() (right after the findByName function): function renderHomeView() { var html = "<h1>Directory</h1>" + "<input class='search-key' type='search' placeholder='Enter name'/>" + "<ul class='employee-list'></ul>"; $('body').html(html); $('.search-key').on('keyup', findByName); } 4. modify the data adapter initialization logic: when the adapter has been initialized, call the renderHomeView() function to programmatically display the Home View: var adapter = new MemoryAdapter(); adapter.initialize().done(function () { renderHomeView(); }); 5. make sure you remove the original event registration in the Event Registration section. 6. remove the click event handler for the help button (in the Event Registration section). 7. test the application.

Use Templates

Writing HTML fragments in JavaScript and programmatically inserting them into the DOM results in: 1. harder to write code 2. harder to maintain code HTML templates address this by decoupling UI (HTML markup) from code. HTML template solutions: e.g. Mustache.js, Handlebars.js, and Underscore.js.

Let's Code! Native Scrolling

To anchor the header at the top of the screen, and scroll the employee list only: 1. in Home template, add css class scroller to div surrounding employee list ul: <div class="topcoat-list__container scroller"> <ul class="topcoat-list list employee-list"></ul> </div> 2. in assets/css/styles.css define the scroller class as follows: .scroller { overflow: auto; -webkit-overflow-scrolling: touch; position: absolute; top: 141px; bottom: 0px; left: 0px; right: 0px; } 3. test the application On iOS, you can still scroll down the web view (including the header). Slow (CPU)

.page { position: absolute;top: 0;left: 0;width: 100%;height: 100%; } .left { left: -100%; } .center { left: 0; } .right { left: 100%; } .transition { transition-duration: .25s; }

Fast (Hardware Acceleration - GPU)

.page {
    position: absolute;top: 0;left: 0;width: 100%;height: 100%;
    transform: translate3d(0, 0, 0);
}
.left {
    transform: translate3d(-100%, 0, 0);
}
.center {
    transform: translate3d(0, 0, 0);
}
.right {
    transform: translate3d(100%, 0, 0);
}
.transition {
    transition-duration: .25s;
}
